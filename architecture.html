Architecture Overview

This page provides an overview of the architecture for the Kubernetes-based network monitoring system. Below is a flowchart that shows how the different components (Pods, services, Prometheus, Grafana, NGINX) interact with each other.

Diagram of Data Flow

The diagram below represents the data flow from the monitoring pod to Prometheus, which collects network traffic metrics. Grafana visualizes the data, while NGINX secures access to the system.

Component Breakdown

  • Monitoring Pod: Captures network traffic using tcpdump. It then sends the data to Prometheus for metrics collection.
  • Prometheus: Scrapes metrics from the monitoring pod and stores it for future queries.
  • Grafana: Visualizes the metrics stored in Prometheus, allowing for real-time data visualization and analysis.
  • NGINX Reverse Proxy: Secures remote access to the system, ensuring that users can only interact with the Kubernetes cluster through a secure entry point.
  • Persistent Volume: Used to store the network traffic data for later backup and analysis.
  • Traffic Backup Pod: Backs up network traffic data from the persistent volume.

Network Security

The NGINX reverse proxy is a critical part of the architecture, as it secures access to the network. All incoming traffic must pass through the reverse proxy, which ensures that only authorized users can access the Kubernetes Dashboard, Grafana, and other services.
