Future Enhancements and Roadmap

This page outlines the planned enhancements and roadmap for the Kubernetes-based network monitoring cluster. Each enhancement is designed to improve functionality, automation, and security in a phased approach.

Planned Enhancements

1. Alerting (30 days)

Goal: Integrate Alertmanager to notify on various types of network, pod, cluster, and security issues.

2. Jenkins Integration (90 days)

Goal: Automate the process of creating pods and updating configurations of existing pods using Jenkins. This will improve the speed and consistency of configuration management.

3. Database Pod Exploration (180 days)

Goal: Investigate the efficiency of using a dedicated database pod (e.g., MySQL or PostgreSQL) for storing network traffic data compared to the current traffic backup pod.

4. Enhancing Security (Ongoing)

Goal: Continuously improve the security of the cluster. Initial steps include reviewing NGINX configurations for vulnerabilities and potentially automating security checks. Further security enhancements will be explored as the project evolves.

Roadmap

The roadmap below shows the planned timeline for each enhancement phase:

  • Phase 1 (30 days): Complete the integration of Alertmanager for comprehensive alerting on network and security issues.
  • Phase 2 (90 days): Implement Jenkins for pod creation and configuration management automation.
  • Phase 3 (180 days): Explore and potentially implement a database pod for more efficient data storage.
  • Ongoing: Continue enhancing security by identifying and addressing potential vulnerabilities.
